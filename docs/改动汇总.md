## 二、按文件详细说明

### 1. php/sql.php

- **目的**：数据库连接配置可维护，便于本地与部署环境切换。
- **改动**：
  - 新增 `DB_HOST`（127.0.0.1）、`DB_USER`（morgan），`DB_DATABASE` 改为 `morgan_db`。
  - `SqlConnectDatabase()` 使用 `mysqli_connect(DB_HOST, DB_USER, DB_PASSWORD)`，不再写死主机与用户名。
- **注意**：密码仍在 `php/_private.php` 的 `DB_PASSWORD` 中配置。

---

### 2. php/sql/sqldailyclose.php

- **目的**：避免传入空日期或空 key 时拼出 `date = ''` 等非法 SQL，引发 MySQL 报错。
- **改动**：
  - `GetFromDate($strKeyId, $strDate, $iNum)`：若 `$strKeyId` 或 `$strDate` 为 false/空串，直接 `return false`。
  - `GetToDate($strKeyId, $strDate, $iNum)`：同上。
  - `GetRecordPrev($strKeyId, $strDate)`：同上。
  - `GetRecord($strKeyId, $strDate)`：增加对 `$strDate === null` 的判断，满足则 `return false`。

- **从入口 `…/woody/res/lofcn.php` 为何会出现日期/key 为空**：
  - **调用链**：`lofcn.php` → `_mystockgroup.php` → `_echoStockGroupArray()` 开头调用 `GetChinaMoney(new CnyReference('USCNY'))` → `chinamoney.php` 中 `_chinaMoneyNeedData($ref->GetDate())` → `$nav_sql->GetRecord(SqlGetStockId('USCNY'), $strDate)`，即 `DailyCloseSql::GetRecord($strKeyId, $strDate)`。
  - **`$strKeyId` 为空**：`SqlGetStockId('USCNY')` 在 **stock 表尚未有 USCNY 代码**（如首次部署、未导入品种）时返回 `false`，导致 key 为空。
  - **`$strDate` 为空**：`$strDate` 来自 `CnyReference('USCNY')->GetDate()`。`MysqlReference::LoadSqlData()` 仅在 `$nav_sql->GetRecordNow($this->strSqlId)` 有记录时给 `$this->strDate` 赋值；若 **净值表里尚无 USCNY 的任何记录**（首次访问、未拉过中国货币网），则 `strDate` 未赋值，`GetDate()` 可能为 `null`/`false`/空，传入 `GetRecord` 即日期为空。
  - **结论**：首次部署或库中尚无汇率数据时，从 LOF 汇总页（lofcn）进入就会走到上述路径，若不先拦截空 key/空日期，会拼出非法 SQL 导致 MySQL 报错；因此在此处做防御性判断是必要的。

---

### 8. php/stock/holdingsref.php

- **目的**：汇率未配置时给出明确提示；汇率取数增加兜底；避免除零与未处理 false。
- **改动概要**：
  - 新增成员 `$bRateNotConfigured`；在 `_estNav()` 中若 `GetAdjustHkd()` 或 `GetAdjustCny()` 为 false，则置为 true 并 return false。
  - 新增 `GetRateNotConfigured()` 供页面判断是否显示「请配置汇率」。
  - 新增 `_getCnyRate($cny_ref, $strDate)`：优先从 ref 取汇率，无效时用 `SqlGetNav` / `SqlGetNavByDate` 直接查库。
  - `GetAdjustHkd()`：持仓日无汇率时用最新价；并增加对 ref 为空时直接查库的兜底；用 `_getCnyRate` 取当前 HKCNY/USCNY；对除零与无效值 return false。
  - `GetAdjustCny()`：用 `_getCnyRate($this->uscny_ref)` 取 USCNY，无效时 return false；保留原有持仓日与除零判断。
  - `GetOfficialNav()`、`GetFairNav()`、`GetRealtimeNav()`、`GetNavChange()`：对 `_estNav()` 返回 false 做判断并相应 return false，避免继续用无效数值。

---

### 9. php/stock/mysqlref.php

- **目的**：中文名从 GBK 转 UTF-8 时优先用 mb_convert_encoding，减少缺字或乱码。
- **改动**：在 `GetEnglishName`、`GetChineseName` 的 `bConvertGB2312` 分支中，在调用 `GbToUtf8` 前先尝试 `mb_convert_encoding(..., 'UTF-8', 'GBK')`，成功则直接返回该结果。

---

### 10. php/ui/fundestparagraph.php

- **目的**：当估值因「汇率未配置」无法计算时，在估值表中明确提示用户。
- **改动**：在 `_echoFundEstTableItem()` 中，若 `GetOfficialNav()` 为 false 且 `GetRateNotConfigured()` 为 true，则官方估值列显示「请配置汇率」（含 title 提示 USCNY/HKCNY），日期列照常，溢价列显示「—」；否则保持原逻辑（价格与涨跌幅）。

---

### 11. php/ui/imagedisp.php

- **目的**：图片文件缺失或非合法 JPEG 时，不调用 `imagesx($imgOrg)`，避免 PHP 8+ 传入 false 导致 TypeError。
- **改动**：`ImgAutoQuote()` 中在 `imagecreatefromjpeg()` 之后判断 `$imgOrg === false`，为真则直接 `return GetBreakElement().GetQuoteElement($strText).GetBreakElement();`，不再执行后续 imagesx/imagesy 及缩放逻辑。

---

### 12. php/url.php

- **目的**：本地访问时 REMOTE_ADDR 可能为 127.0.0.1 或 ::1，需允许通过；同时保持对 X-Forwarded-For/Client-IP 的严格校验。
- **改动**：
  - 新增 `filter_valid_ip_allow_private($strIp)`：仅用 `FILTER_VALIDATE_IP`，允许私有与 IPv6。
  - `UrlGetIp()`：对 `REMOTE_ADDR` 使用 `filter_valid_ip_allow_private`；对 X-Forwarded-For、Client-IP 仍用 `filter_valid_ip`；结构改为多个 if 分支，避免本地访问报「Unknown IP」。

---

### 13. woody/res/php/_mystockgroup.php

- **目的**：打开 LOF 汇总页时即尝试拉取当日 USCNY/HKCNY，减少估值表出现「请配置汇率」；且不因汇率未配置而减少估值表展示条数。
- **改动**：
  - 增加 `require_once('../../php/stock/chinamoney.php');`
  - 在 `_echoStockGroupArray()` 开头调用 `GetChinaMoney(new CnyReference('USCNY'));`，在构建估值表前拉取汇率。
  - 基金入表条件由 `if ($fund->GetOfficialNav())` 改为 `if (RefHasData($fund))`：有数据即展示一行，估值算不出时该行显示「请配置汇率」，避免整项从表中消失导致「数量大量减少」。
